# HR-бот для скрининга кандидатов в ML-команду

Учебный проект по курсу «Технологии глубокого обучения и больших языковых моделей».  
Бот проводит первичное интервью на одну из пяти ролей в ML-проекте:

- Project Manager
- Data Analyst
- Data Engineer
- Data Scientist
- MLOps Engineer

После диалога (роль → опыт → навыки → ожидания → короткий квиз) бот выдаёт вердикт: «подходит / не подходит» через кастомное действие.

Репозиторий на GitHub: [esheshka/rasa_hr_project](https://github.com/esheshka/rasa_hr_project)

---

## Архитектурные особенности

### Один ответ пользователя → одна реплика бота

В `config.yml` отключены TEDPolicy и MemoizationPolicy, остаётся только RulePolicy и формы.  
Сделано так, чтобы не было конкурирующих предсказателей за один вход и не терялся `active_loop` из-за усечённых историй в Memoization.

### Длинное приветствие — один шаблон

Всё приветствие собрано в `utter_greet_full` и вызывается через `action_greet_user`.  
Никаких лишних сообщений подряд.

### Пустой ответ после формы — проблема с action-сервером

Если после вопроса формы бот ничего не отвечает — скорее всего, не запущен `rasa run actions` или недоступен порт 5055.

## Установка и запуск

### Подготовка окружения

Рекомендуется Python 3.10 (как в лекциях).

```bash
cd rasa_hr_project
python3.10 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

### Запуск action-сервера (терминал 1)

```bash
source .venv/bin/activate
rasa run actions
```

### Запуск бота (терминал 2)

```bash
source .venv/bin/activate
rasa train
rasa shell
```

Полезные команды по ходу дела:

```bash
rasa data validate
rasa shell nlu
```

## Сценарий диалога

1. Пользователь здоровается или спрашивает «какие роли есть?»
2. Пишет «хочу пройти интервью» — запускается форма `interview_form`
3. Бот последовательно собирает:
   - целевую роль (или «не знаю» — тогда автоподбор по навыкам)
   - стаж (годы / «нет опыта»)
   - навыки и технологии
   - ожидания по зарплате и формату работы
4. После формы вызывается `action_evaluate_candidate`:
   - эвристика по стажу и ключевым словам
   - если на шаге роли было «не знаю» — бот сам подбирает роль по тексту навыков
5. Если кандидат подходит → вердикт с рекомендацией  
   Если нет → отказ с пояснением

Во время формы бот обрабатывает:
- out_of_scope — остаётся в форме
- «отмена интервью» и «прощание» — снимает форму, завершает диалог

После завершения или прерывания слот `intro_done` сбрасывается, следующее «Здравствуйте» снова покажет полное приветствие.

Чтобы сбросить состояние вручную: `/restart`

## Автоматическая проверка сценариев (REST)

После `rasa train` поднимите два сервера:

```bash
rasa run actions --port 5055

rasa run --enable-api --cors "*" -p 5006
```

В третьем терминале запустите скрипт:

```bash
python scripts/e2e_dialog.py
```

Код возврата 0 — все сценарии прошли.

Скрипт проверяет:
- приветствие и hi
- fallback (непонятные фразы)
- список ролей
- полный happy-path (успешное интервью)
- ветку «не знаю роль» с автоподбором по навыкам
- отказ по слабым навыкам
- прощание
- out_of_scope
- прерывание формы (например, анекдот)
- длинное сообщение

## Структура проекта

| Файл | Назначение |
|------|------------|
| `domain.yml` | Интенты, сущность `job_role`, слоты, форма, шаблоны ответов |
| `data/nlu.yml` | Примеры фраз для интентов, сущности, lookup-таблицы ролей |
| `data/rules.yml` | Правила: приветствие, список ролей, старт формы, завершение |
| `data/stories.yml` | Минимальные истории для FormValidationAction (обучение без TED) |
| `actions/actions.py` | Логика оценки кандидата и прерывания формы |
| `scripts/e2e_dialog.py` | Автоматический прогон сценариев через REST API |

---

## Если что-то не работает

После вопроса формы бот молчит → скорее всего, не запущен `rasa run actions` или порт 5055 занят. Проверить `endpoints.yml`.

Бот отвечает невпопад → попробовать обучить модель заново: `rasa train`

Сбросить разговор в консоли → `/restart`
